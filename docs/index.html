What is Genius Team?
An agent orchestration framework
Genius Team is not a task runner ‚Äî it's a framework that spawns specialized agents for each phase of your project, keeps memory across sessions, and gates every key decision behind a human checkpoint.

üé§ Phase 1: Interview

Every project starts with genius-interviewer. It asks the right questions before any work begins ‚Äî no assumptions, no drift. Output: a validated brief, approved by you.

üìã Phase 2: Ideation

Specialized agents spawn for market analysis, specs, design, and architecture. Each phase ends with a human checkpoint ‚Äî you approve before the next phase launches.

‚ö° Phase 3: Execution

Agent Teams execute in parallel with git worktree isolation and mandatory QA loops. A Retrospective Engine runs after each phase ‚Äî feeding learnings back into memory.

üß† Cross-Project Memory

Every decision, every lesson, every trade-off is stored. New projects inherit wisdom from past ones. The more you build, the smarter your agents become.

FAQ & Troubleshooting
Common questions

‚ùì I get "command not found: claude"

Claude Code CLI isn't installed. Run: curl -fsSL https://claude.ai/install.sh | bash then restart your terminal. For Codex: npm install -g @openai/codex.

‚ùì I get "permission denied"

The script needs execute permission. Try: chmod +x scripts/setup.sh or simply run with bash scripts/setup.sh

‚ùì What is a terminal?

It's the app where you type commands. On Mac: search "Terminal". On Windows: use WSL or Git Bash. On VS Code: press Ctrl+` to open it.

‚ùì The command says "my-project" but I want a different name

Replace "my-project" with YOUR project name! Example: ...create.sh) booking-app --engine=claude. No spaces, use dashes.

‚ùì Which engine should I choose?

Claude Code if you already use Claude. Codex CLI if you prefer OpenAI. Dual Mode if you want both ‚Äî Claude builds, Codex reviews, they challenge each other for the best result.

‚ùì How does Dual Mode work?

Claude Code acts as Builder (implements features), while Codex CLI acts as Challenger (reviews and critiques). They iterate via shared files in .genius/ until both agree the code is solid.

‚ùì AI seems stuck or keeps repeating

Run /guard-check to see if something went wrong. If needed, /guard-recover gets you back on track. Works with both Claude and Codex.

‚ùì I closed the terminal and lost my progress

No worries! Just cd back to your project folder, run claude (or codex), then /continue. The memory system in .genius/ remembers everything.

‚ùì Do I need a paid subscription?

Yes ‚Äî Claude Code requires a Claude Pro or API subscription (claude.ai/pricing). Codex CLI requires an OpenAI account. Both have free trials.

‚ùì How do I update to a new version?

Run /genius-upgrade inside your project. Or from the terminal: bash <(curl -fsSL .../scripts/upgrade.sh)
