# Genius Autoresearch v22.0 — Autonomous Optimization Loop

**Karpathy's autoresearch pattern applied to any project component. Modify → Test → Measure → Keep if better → Repeat.**

## Running with `/loop`

For continuous autonomous optimization, use Claude Code's native `/loop` command:

```
/loop 5m /skill genius-autoresearch
```

The skill self-paces iterations. For overnight optimization, combine with safe stop conditions (max iterations, max time, metric target).

Self-paced variant (skill picks next wake-up):
```
/loop /skill genius-autoresearch
```

## HOW IT WORKS

### Phase 1: Discovery (Interactive — user guides)

When /autoresearch is invoked:

1. **Ask: "What do you want to optimize?"**
   Examples the user might say:
   - "The routing accuracy of genius-team skill"
   - "The performance of the API endpoints"
   - "The quality of specs generated by genius-specs"
   - "The conversion rate of the landing page"
   - "The anti-drift detection rate"

2. **Ask: "How do we measure 'better'?"**
   Guide the user to define a clear metric:
   - Routing accuracy: "% of test prompts routed to the correct skill"
   - Performance: "Response time in ms on 10 test requests"
   - Spec quality: "Score 1-10 on completeness, coherence, actionability"
   - Conversion: "Click-through rate on CTA"
   - Anti-drift: "% of drift scenarios detected"

3. **Ask: "What are the constraints?"**
   - Max iterations? (default: 30)
   - Time budget per iteration? (default: 5 min)
   - What must NOT change? (safety constraints)
   - Minimum acceptable score to keep a change?

4. **Ask: "What file(s) should I modify?"**
   - The target file(s) that will be changed each iteration
   - Example: `SKILL.md`, `CLAUDE.md`, `scripts/*.sh`, `src/*.ts`

5. **Confirm the plan:**
   ```
   AUTORESEARCH PLAN
   ═══════════════════
   Target: [what's being optimized]
   Metric: [how we measure]
   File(s): [what gets modified]
   Iterations: [max N]
   Budget: [time per iteration]
   Constraints: [what must not change]
   
   Ready to start? (y/n)
   ```

### Phase 2: Baseline (Automatic)

1. **Run the metric** on the current version → baseline score
2. **Save a backup** of the target file(s)
3. **Log baseline** to `.genius/autoresearch/research-log.jsonl`:
   ```json
   {"iteration": 0, "type": "baseline", "score": 7.2, "timestamp": "ISO", "changes": "none"}
   ```

### Phase 3: Iteration Loop (Autonomous)

For each iteration (1 to max):

1. **Analyze** what could be improved based on:
   - Previous iteration results
   - The metric definition
   - Patterns from observations/corrections
   - What worked and what didn't in prior iterations

2. **Modify** the target file(s) — one focused change per iteration
   - Document WHAT changed and WHY in the log

3. **Test** — run the measurement
   - If the metric needs human judgment (quality score) → ask the user
   - If the metric is automated (test pass rate, timing) → run it

4. **Compare** to the best score so far:
   - **Better** → Keep the change. Log: `{"iteration": N, "score": X, "kept": true, "change": "..."}`
   - **Same** → Keep if simpler/cleaner, otherwise revert
   - **Worse** → Revert the change. Log: `{"iteration": N, "score": X, "kept": false, "change": "...", "reason": "score decreased"}`

5. **Report** progress:
   ```
   Iteration 7/30 — Score: 8.4 (+1.2 from baseline)
   Change: [what was tried]
   Result: ✅ Kept (best so far)
   ```

6. **Stop conditions:**
   - Score reaches 9.5+ → "Near-optimal. Stopping."
   - 5 consecutive iterations with no improvement → "Plateau reached. Stopping."
   - Max iterations reached
   - User says STOP

### Phase 4: Report (Automatic)

After stopping:

1. **Generate summary** to `.genius/autoresearch/report-[topic]-[date].md`:
   ```markdown
   # Autoresearch Report: [Topic]
   
   ## Results
   - Baseline: [score]
   - Final: [score] (+[delta])
   - Iterations: [N] run, [kept] kept, [reverted] reverted
   
   ## Best Changes
   1. [Change that improved most] (+X)
   2. [Second best change] (+Y)
   
   ## Failed Attempts
   1. [What was tried and why it didn't work]
   
   ## Recommendation
   [Final recommendation for the user]
   ```

2. **Generate playground** if applicable:
   `open .genius/outputs/autoresearch-playground.html`

3. **Ask user**: "Satisfied with the result? Or should I continue with more iterations?"

4. **Log to memory**: Append decisions to `.genius/memory/decisions.json`

## RULES

- **ALWAYS backup** target files before starting
- **NEVER modify** files outside the declared target(s) without asking
- **ONE change per iteration** — don't batch multiple changes
- **REVERT completely** on worse score — no partial keeps
- **LOG everything** — even failed attempts are valuable data
- **STOP on plateau** — don't waste iterations when stuck
- If the user needs to judge quality → show them the output and ASK for a score
- Never auto-score subjective quality — always ask the user

## Definition of Done
- [ ] User confirmed the target, metric, and constraints
- [ ] Baseline measured and logged
- [ ] At least 5 iterations completed
- [ ] Final score > baseline
- [ ] Report generated
- [ ] User validated the final result

## Handoff
After autoresearch → show report, ask user if satisfied. If yes → done. If no → suggest continuing or changing approach.
